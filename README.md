# LightingManagementGUI

イベントや舞台で使用する照明機材について、使用可能在庫、現場ごとの使用機材、故障・修理状況を管理するJava Swing製デスクトップアプリケーションです。

職業訓練で学んだJavaの復習と、前職で経験した照明機材準備の課題を題材に制作しました。

## 制作背景

前職では、イベントに必要な照明機材と数量をExcelの一覧表で管理していました。使用しない機材を含む大きな表から必要な情報を探す必要があり、業務に慣れていない時期は、目的の機材を探す作業や準備後の確認に時間がかかっていました。

経験を積めば見つけやすくなる一方で、経験の少ない担当者でも必要な情報を判断しやすくする方法があれば、準備時間と心理的な負担を軽減できるのではないかと考えました。この経験が、機材管理を題材にしたアプリケーションを制作するきっかけになりました。

## 制作目的

イベントで使用する機材だけを現場ごとにまとめ、準備や確認に必要な情報を見つけやすくすることを目的として制作しました。

具体的には、次のことを目指しています。

- 機材をカテゴリ別に探せるようにする
- 現場ごとに必要な機材だけを登録する
- 使用可能な在庫数を把握する
- ピッキングに必要な情報だけを表示・印刷する
- 故障中と修理済みの機材を管理する

本作は、前職で感じた課題を題材として、要件整理から画面設計、実装、動作確認まで取り組んだ初めてのJavaアプリケーションです。

## 主な機能

### 現場管理

- 現場名と日付を入力して現場を作成
- 作業対象の現場をコンボボックスから選択
- 3行×3列のカテゴリ別一覧から機材を登録
- 選択中の現場に登録した機材を削除
- 現場一覧の確認
- 現場詳細・ピッキングリストのプレビューと印刷

### 在庫管理

- 機材ごとの使用可能在庫を表示
- カテゴリ別一覧から機材を選択して在庫補充
- 現場への登録時に使用可能在庫を減算
- 現場からの削除時に使用可能在庫へ返却

### 故障・修理管理

- データ管理画面の機材一覧から故障機材を登録
- 故障機材だけをカテゴリ別画面で表示
- 故障機材一覧から対象を選択して修理完了
- 使用可能在庫と故障在庫を同時に表示

### 操作ログ

画面下部の「操作ログ・登録状況」には、次の情報が表示されます。

- 選択中の現場に登録済みの機材と数量
- 現場への機材登録
- 現場機材削除
- 在庫補充
- 故障登録
- 修理完了

メイン画面と各プレビュー画面は、ディスプレイの利用可能領域に合わせて、最大化操作をしなくても内容を確認しやすいサイズで開きます。

## 基本操作

### 1. 現場を作成する

「新しい現場を作成」で現場名と日付を入力し、「現場追加」を押します。同じ名前の現場は登録できません。

### 2. 作業する現場を選ぶ

「作業する現場」のコンボボックスから対象を選択します。選択すると、画面下部へ登録済みの機材と数量が表示されます。

### 3. 現場へ機材を登録する

カテゴリ別一覧で機材をダブルクリックし、数量を入力します。登録数量が在庫から減り、選択中の現場へ追加されます。

### 4. 登録内容を確認する

- 「現場詳細」：選択中の現場をカテゴリ別に表示
- 「ピッキングリスト表示」：印刷可能なカテゴリ別プレビューを表示
- 「現場一覧」：登録されている現場をログ欄へ表示

プレビュー画面はディスプレイの利用可能領域に合わせて、自動的に見やすいサイズで開きます。

### 5. 現場から機材を削除する

「現場機材削除」を押し、選択中の現場に登録されている機材をカテゴリ別一覧からダブルクリックします。確認後に削除され、数量は使用可能在庫へ戻ります。

### 6. 在庫を補充する

「在庫補充」を押し、カテゴリ別一覧から機材をダブルクリックして補充数を入力します。

### 7. 故障機材を登録する

「データ管理」タブで機材をダブルクリックし、故障数を入力します。入力数が使用可能在庫から故障在庫へ移動します。

上部の「故障機材登録」ボタンから機材名を入力して登録することもできます。

### 8. 故障一覧・修理完了

- 「故障機材一覧」：故障中の機材をカテゴリ別に表示
- 「修理完了」：カテゴリ別一覧から機材をダブルクリックし、修理台数を入力

修理した数量は故障在庫から減り、使用可能在庫へ戻ります。

## 在庫の増減

| 操作 | 使用可能在庫 | 故障在庫 |
| --- | ---: | ---: |
| 現場へ機材登録 | 登録数分減る | 変化なし |
| 現場機材削除 | 削除数分戻る | 変化なし |
| 在庫補充 | 補充数分増える | 変化なし |
| 故障機材登録 | 故障数分減る | 故障数分増える |
| 修理完了 | 修理数分増える | 修理数分減る |

数量には1以上の整数だけを入力できます。在庫または故障在庫を超える操作は実行されません。

## 保存・読込

「データ管理」タブの「保存」「読込」を使用します。

データは、アプリケーションを起動したカレントディレクトリにある次の3ファイルへ保存されます。

| ファイル | 内容 |
| --- | --- |
| `equipments.txt` | 機材名、カテゴリ、使用可能在庫、故障在庫 |
| `projects.txt` | 現場名、日付 |
| `requestItems.txt` | 現場名、機材名、数量 |

ファイルはカンマ区切りで、見出し行はありません。機材名、カテゴリ名、現場名、日付へカンマを含めることは想定していません。

現状は保存先を選択する機能や自動保存を実装していません。保存・読込の操作性は今後の改善項目です。

- Java 21

## 起動方法

### Eclipse

1. Eclipseへプロジェクトをインポートします。
2. `src/Main.java` を開きます。
3. Javaアプリケーションとして実行します。

### コマンドライン

プロジェクトのルートディレクトリで実行します。

```powershell
javac -encoding UTF-8 -d bin src\*.java
java -cp bin Main
```

エントリーポイントは `Main` です。`LightingManagementGUI` は旧画面を起動するクラスであり、現在の管理画面では使用しません。

## クラス構成

| クラス | 役割 |
| --- | --- |
| `Main` | Swing画面、イベント処理、入力検証、在庫更新、保存・読込、プレビュー、印刷 |
| `Equipment` | 機材名、カテゴリ、使用可能在庫、故障在庫 |
| `Project` | 現場名、日付、現場へ登録した機材明細 |
| `RequestItem` | 現場へ登録した機材と数量 |
| `LightingManagementGUI` | 初期開発時の旧画面 |

データは`ArrayList`とSwingのモデルでメモリ上に保持し、データベースは使用していません。

- SPOT
- DIMMER
- PIN SPOT

- HAZE MACHINE
- OTHER

## 開発で工夫した点

### 必要な機材だけを確認できる表示

大量の機材一覧から探す負担を減らすため、現場ごとに登録した機材だけを現場詳細やピッキングリストへ表示できるようにしました。

### カテゴリ別の機材選択

機材名を文字入力して検索する操作を減らし、9カテゴリを3行×3列で表示して、目的の機材を分類から選択できるようにしました。在庫補充、現場機材削除、故障登録、修理完了でも同じ操作方法を使用しています。

### 在庫状況の連動

現場への登録、現場からの削除、故障登録、修理完了に合わせて、使用可能在庫と故障在庫が連動して増減します。在庫数を超える登録や、故障在庫を超える修理完了は実行できません。

### 操作状況の表示

選択中の現場へ登録されている機材と数量に加え、在庫補充や故障登録などの操作結果を画面下部へ表示し、現在の状態を確認しやすくしました。

### 画面サイズの自動調整

メイン画面、現場詳細、ピッキングリスト、故障機材一覧、修理完了画面は、使用しているディスプレイの利用可能領域から表示サイズを決定します。長い機材名も、できるだけ横スクロールせず確認できるようにしています。

## AIの利用

題材と要件は、前職での経験をもとに考えました。初めて企画・開発したアプリケーションのため、実装方法の検討、コードレビュー、不具合調査、画面改善、READMEの整理に生成AIを開発支援として利用しています。

提案された内容は、実際にアプリケーションを起動して確認し、意図と異なる場合は修正を重ねています。

## 現在の課題

- 保存先を利用者が選択できず、自動保存もない
- データを3つのテキストファイルで管理している
- カンマを含む名称を保存できない
- プレビュー印刷は1ページのみで、複数ページへの分割がない
- 画面処理とデータ処理の多くが`Main`へ集約されている
- 実行可能JARやWindowsアプリとしての配布は未対応
